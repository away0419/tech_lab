## 변수 선언 및 초기화

<summary>변수의 개념과 사용법</summary>

### 변수란?

- **정의**: 데이터를 저장하는 메모리 공간에 붙인 이름
- **용도**: 프로그램 실행 중 값을 저장하고 참조
- **특징**: 변수는 값을 변경할 수 있음

### 변수 선언 방법

#### 기본 문법

```java
// 변수 타입 변수명;
int number;
String name;

// 선언과 동시에 초기화
int age = 25;
String greeting = "Hello";
```

#### 변수명 규칙

1. **문자, 숫자, 언더스코어(\_), 달러($) 사용 가능**

   ```java
   int age;           // 올바름
   int _count;        // 올바름
   int $price;        // 올바름 (권장하지 않음)
   int user_name;     // 올바름
   ```

2. **숫자로 시작 불가**

   ```java
   int 1number;       // 오류!
   int number1;        // 올바름
   ```

3. **예약어 사용 불가**

   ```java
   int class;         // 오류! (예약어)
   int public;        // 오류! (예약어)
   int myClass;       // 올바름
   ```

4. **대소문자 구분**

   ```java
   int age;
   int Age;           // 다른 변수
   int AGE;           // 또 다른 변수
   ```

5. **명명 규칙 (관례)**
   - **카멜 케이스 (camelCase)**: 변수명, 메서드명
     ```java
     int studentAge;
     String userName;
     ```
   - **파스칼 케이스 (PascalCase)**: 클래스명
     ```java
     class StudentInfo { }
     ```
   - **상수**: 대문자와 언더스코어
     ```java
     final int MAX_SIZE = 100;
     ```

### 케이스 스타일 (Case Styles) 종류

#### 1. 카멜 케이스 (camelCase)

- **정의**: 첫 단어는 소문자, 이후 단어의 첫 글자는 대문자
- **예시**: `studentName`, `calculateTotal`, `getUserInfo`
- **Java 사용처**: 변수명, 메서드명, 매개변수명
- **특징**: 단어 사이에 구분자 없음

```java
// 변수명
String userName;
int studentCount;
boolean isActive;

// 메서드명
public void calculateTotal() { }
public String getUserName() { }
```

#### 2. 파스칼 케이스 (PascalCase) / 대문자 카멜 케이스 (UpperCamelCase)

- **정의**: 모든 단어의 첫 글자가 대문자
- **예시**: `StudentInfo`, `UserAccount`, `DatabaseConnection`
- **Java 사용처**: 클래스명, 인터페이스명, 생성자명
- **특징**: 카멜 케이스와 유사하지만 첫 글자도 대문자

```java
// 클래스명
class StudentInfo { }
class UserAccount { }

// 인터페이스명
interface Drawable { }
interface Runnable { }
```

#### 3. 스네이크 케이스 (snake_case)

- **정의**: 모든 단어를 소문자로, 단어 사이는 언더스코어(\_)로 구분
- **예시**: `user_name`, `student_count`, `calculate_total`
- **Java 사용처**: 일반적으로 사용하지 않음 (다른 언어에서 주로 사용)
- **특징**: Python, C 등에서 주로 사용

```java
// Java에서는 권장하지 않음
// int user_name;  // 피해야 함
// int student_count;  // 피해야 함
```

#### 4. 대문자 스네이크 케이스 (UPPER_SNAKE_CASE) / SCREAMING_SNAKE_CASE

- **정의**: 모든 글자를 대문자로, 단어 사이는 언더스코어(\_)로 구분
- **예시**: `MAX_SIZE`, `DEFAULT_USER_NAME`, `TIMEOUT_SECONDS`
- **Java 사용처**: 상수명 (final 변수)
- **특징**: Java에서 상수를 나타낼 때 사용

```java
// 상수명
final int MAX_SIZE = 100;
final String DEFAULT_USER_NAME = "Guest";
public static final int TIMEOUT_SECONDS = 30;
```

#### 5. 케밥 케이스 (kebab-case) / 하이픈 케이스 (hyphen-case)

- **정의**: 모든 단어를 소문자로, 단어 사이는 하이픈(-)으로 구분
- **예시**: `user-name`, `student-count`, `calculate-total`
- **Java 사용처**: 사용 불가 (하이픈은 연산자로 인식)
- **특징**: HTML, CSS, URL 등에서 주로 사용

```java
// Java에서는 사용 불가!
// int user-name;  // 오류! 하이픈은 빼기 연산자
```

#### 6. 헝가리안 표기법 (Hungarian Notation)

- **정의**: 변수 타입을 접두사로 붙이는 방식
- **예시**: `strUserName`, `intCount`, `bIsActive`
- **Java 사용처**: 권장하지 않음 (구식 방식)
- **특징**: C/C++에서 주로 사용, Java에서는 피해야 함

```java
// Java에서는 권장하지 않음
// String strUserName;  // 피해야 함
// int intCount;  // 피해야 함
// boolean bIsActive;  // 피해야 함

// 대신 의미 있는 이름 사용
String userName;  // 좋은 예
int count;  // 좋은 예
boolean isActive;  // 좋은 예
```

### 케이스 스타일 비교표

| 케이스 스타일        | 예시          | Java 사용처        | 비고               |
| -------------------- | ------------- | ------------------ | ------------------ |
| **camelCase**        | `userName`    | 변수, 메서드       | 가장 많이 사용     |
| **PascalCase**       | `UserName`    | 클래스, 인터페이스 | 첫 글자 대문자     |
| **snake_case**       | `user_name`   | 사용 안 함         | Python, C에서 사용 |
| **UPPER_SNAKE_CASE** | `MAX_SIZE`    | 상수               | final 변수         |
| **kebab-case**       | `user-name`   | 사용 불가          | HTML, CSS에서 사용 |
| **Hungarian**        | `strUserName` | 권장 안 함         | 구식 방식          |

### Java에서의 케이스 스타일 사용 가이드

#### 권장 사용법

```java
// ✅ 카멜 케이스: 변수, 메서드
String userName;
int studentCount;
public void calculateTotal() { }

// ✅ 파스칼 케이스: 클래스, 인터페이스
class StudentInfo { }
interface Drawable { }

// ✅ 대문자 스네이크 케이스: 상수
final int MAX_SIZE = 100;
public static final String DEFAULT_NAME = "Guest";
```

#### 피해야 할 사용법

```java
// ❌ 스네이크 케이스: Java에서는 사용 안 함
// String user_name;  // 피해야 함
// int student_count;  // 피해야 함

// ❌ 케밥 케이스: Java에서는 사용 불가
// String user-name;  // 오류! 하이픈은 연산자

// ❌ 헝가리안 표기법: Java에서는 권장 안 함
// String strUserName;  // 피해야 함
// int intCount;  // 피해야 함
```

#### C/C++

### 케이스 스타일 선택 가이드

#### Java 프로젝트에서

1. **변수/메서드**: 항상 카멜 케이스
2. **클래스/인터페이스**: 항상 파스칼 케이스
3. **상수**: 항상 대문자 스네이크 케이스
4. **패키지**: 모두 소문자, 점(.) 구분

#### 일관성 유지

- 프로젝트 전체에서 동일한 스타일 유지
- 팀 내에서 합의된 규칙 따르기
- 기존 코드베이스의 스타일 준수

### Java 명명 규칙 (Naming Convention) 상세

#### 1. 패키지명 (Package)

- **규칙**: 모두 소문자, 단어 구분은 점(.)
- **예시**:
  ```java
  package com.example.myproject;
  package java.util;
  package org.apache.commons;
  ```
- **주의사항**: 숫자로 시작 불가, 예약어 사용 불가

#### 2. 클래스명 (Class)

- **규칙**: 파스칼 케이스 (PascalCase), 명사 사용
- **예시**:
  ```java
  class Student { }
  class UserAccount { }
  class DatabaseConnection { }
  ```
- **특수 케이스**:
  ```java
  class MyClass { }        // 일반 클래스
  abstract class Animal { } // 추상 클래스
  final class Constants { } // final 클래스
  ```

#### 3. 인터페이스명 (Interface)

- **규칙**: 파스칼 케이스 (PascalCase), 형용사 또는 명사
- **예시**:
  ```java
  interface Drawable { }
  interface Runnable { }
  interface List { }
  interface Comparable { }
  ```
- **관례**: `-able`, `-ible`로 끝나는 형용사 형태 많이 사용

#### 4. 메서드명 (Method)

- **규칙**: 카멜 케이스 (camelCase), 동사로 시작
- **예시**:
  ```java
  public void calculateTotal() { }
  public String getUserName() { }
  public boolean isValid() { }
  public void setAge(int age) { }
  ```
- **명명 패턴**:
  - **getter**: `get + 필드명` (boolean은 `is + 필드명`)
    ```java
    public String getName() { }
    public boolean isActive() { }
    ```
  - **setter**: `set + 필드명`
    ```java
    public void setName(String name) { }
    ```
  - **boolean 반환**: `is`, `has`, `can` 등으로 시작
    ```java
    public boolean isEmpty() { }
    public boolean hasPermission() { }
    public boolean canEdit() { }
    ```

#### 5. 변수명 (Variable)

- **규칙**: 카멜 케이스 (camelCase), 명사 사용
- **예시**:
  ```java
  int studentCount;
  String userName;
  double totalPrice;
  boolean isActive;
  ```
- **변수 타입별**:
  - **지역 변수**: `count`, `name`, `result`
  - **인스턴스 변수**: `studentName`, `accountBalance`
  - **클래스 변수**: `MAX_SIZE`, `DEFAULT_VALUE` (상수는 대문자)

#### 6. 상수명 (Constant)

- **규칙**: 모두 대문자, 단어 구분은 언더스코어(\_)
- **예시**:
  ```java
  final int MAX_SIZE = 100;
  final String DEFAULT_USER_NAME = "Guest";
  final double PI = 3.14159;
  public static final int TIMEOUT_SECONDS = 30;
  ```
- **주의**: `static final` 조합이 일반적

#### 7. 매개변수명 (Parameter)

- **규칙**: 카멜 케이스 (camelCase), 의미 있는 이름
- **예시**:
  ```java
  public void setUser(String userName, int userAge) { }
  public void calculate(int firstNumber, int secondNumber) { }
  ```
- **나쁜 예**:
  ```java
  public void method(int a, int b) { }  // 의미 불명확
  ```

#### 8. 생성자명 (Constructor)

- **규칙**: 클래스명과 동일 (파스칼 케이스)
- **예시**:
  ```java
  public class Student {
      public Student() { }  // 기본 생성자
      public Student(String name) { }  // 매개변수 생성자
  }
  ```

#### 9. 예외 클래스명 (Exception)

- **규칙**: 파스칼 케이스, `Exception`으로 끝남
- **예시**:
  ```java
  class MyException extends Exception { }
  class InvalidInputException extends Exception { }
  class FileNotFoundException extends Exception { }
  ```

#### 10. 열거형명 (Enum)

- **규칙**: 파스칼 케이스 (클래스와 동일)
- **예시**:
  ```java
  enum Color { RED, GREEN, BLUE }
  enum DayOfWeek { MONDAY, TUESDAY, WEDNESDAY }
  ```
- **열거형 상수**: 모두 대문자, 언더스코어 구분
  ```java
  enum Status { ACTIVE, INACTIVE, PENDING_APPROVAL }
  ```

### 명명 규칙 요약표

| 요소            | 규칙                     | 예시                  |
| --------------- | ------------------------ | --------------------- |
| **패키지**      | 모두 소문자, 점(.) 구분  | `com.example.project` |
| **클래스**      | 파스칼 케이스            | `StudentInfo`         |
| **인터페이스**  | 파스칼 케이스            | `Drawable`            |
| **메서드**      | 카멜 케이스, 동사        | `calculateTotal()`    |
| **변수**        | 카멜 케이스, 명사        | `studentName`         |
| **상수**        | 모두 대문자, 언더스코어  | `MAX_SIZE`            |
| **매개변수**    | 카멜 케이스              | `userName`            |
| **생성자**      | 클래스명과 동일          | `Student()`           |
| **예외**        | 파스칼 케이스, Exception | `MyException`         |
| **열거형**      | 파스칼 케이스            | `Color`               |
| **열거형 상수** | 모두 대문자              | `RED`, `GREEN`        |

### 명명 규칙 모범 사례

#### 좋은 예

```java
// 클래스명: 명확하고 구체적
class UserAccount { }
class DatabaseConnection { }

// 메서드명: 동작을 명확히 표현
public void calculateTotalPrice() { }
public boolean isValidEmail(String email) { }

// 변수명: 의미 있는 이름
int studentCount;
String userEmailAddress;
boolean hasPermission;

// 상수명: 명확한 의미
final int MAX_RETRY_COUNT = 3;
final String DEFAULT_LANGUAGE = "ko";
```

#### 나쁜 예

```java
// 클래스명: 모호함
class Data { }  // 무엇의 데이터인지 불명확
class Manager { }  // 무엇을 관리하는지 불명확

// 메서드명: 동작이 불명확
public void process() { }  // 무엇을 처리하는지?
public void do() { }  // 너무 일반적

// 변수명: 의미 불명확
int a, b, c;  // 무엇을 나타내는지?
String temp;  // 임시 변수명은 피하기
String data1, data2;  // 숫자로 구분하는 것은 나쁜 습관

// 상수명: 규칙 위반
final int maxSize = 100;  // 대문자 사용해야 함
final int MAXSIZE = 100;  // 언더스코어 필요
```

### 추가 명명 가이드라인

#### 1. 약어 사용

- **원칙**: 일반적으로 알려진 약어만 사용
- **좋은 예**:
  ```java
  String id;  // ID는 일반적으로 알려짐
  String url;  // URL은 일반적으로 알려짐
  int maxValue;  // max는 일반적으로 알려짐
  ```
- **나쁜 예**:
  ```java
  String usrNm;  // userName으로 쓰는 것이 좋음
  int numOfStds;  // numberOfStudents로 쓰는 것이 좋음
  ```

#### 2. 불리언 변수명

- **규칙**: `is`, `has`, `can`, `should` 등으로 시작
- **예시**:
  ```java
  boolean isActive;
  boolean hasPermission;
  boolean canEdit;
  boolean shouldRetry;
  ```

#### 3. 컬렉션 변수명

- **규칙**: 복수형 사용
- **예시**:
  ```java
  List<String> students;
  Map<String, Integer> scores;
  Set<String> uniqueNames;
  ```

#### 4. 임시 변수

- **원칙**: 가능한 한 피하고, 사용 시 의미 있는 이름 사용
- **나쁜 예**:
  ```java
  String temp;
  int x, y, z;
  ```
- **좋은 예**:
  ```java
  String currentUserName;
  int firstNumber, secondNumber;
  ```

### 변수 초기화

#### 초기화 방법

```java
// 1. 선언만 하기 (지역 변수는 반드시 초기화 필요)
int number;
// System.out.println(number);  // 오류! 초기화되지 않음

// 2. 선언과 동시에 초기화
int age = 25;

// 3. 여러 변수 동시 선언
int a = 10, b = 20, c = 30;

// 4. 나중에 초기화
int count;
count = 100;
```

#### 변수 스코프 (범위)

```java
public class VariableScope {
    int instanceVar = 10;  // 인스턴스 변수 (클래스 전체)

    public void method() {
        int localVar = 20;  // 지역 변수 (메서드 내부만)

        if (true) {
            int blockVar = 30;  // 블록 변수 (if 블록 내부만)
            System.out.println(blockVar);
        }
        // System.out.println(blockVar);  // 오류! 블록 밖에서 접근 불가
    }
}
```

### 변수 타입별 특징

#### 기본형 변수 (Primitive)

```java
int number = 10;  // 값 자체가 저장됨
int copy = number;  // 값 복사
copy = 20;  // number는 여전히 10
```

#### 참조형 변수 (Reference)

```java
String str = "Hello";  // 객체의 주소가 저장됨
String copy = str;  // 주소 복사
// str과 copy는 같은 객체를 참조
```

## 기본 데이터 타입 (Primitive Types)

<details>
<summary>8가지 기본 데이터 타입 상세 설명</summary>

### 기본 데이터 타입 개요

Java는 8가지 기본 데이터 타입을 제공합니다:

1. **정수형**: byte, short, int, long
2. **실수형**: float, double
3. **문자형**: char
4. **논리형**: boolean

### 1. 정수형 (Integer Types)

#### byte

- **크기**: 1바이트 (8비트)
- **범위**: -128 ~ 127
- **용도**: 작은 정수, 바이너리 데이터 처리

```java
byte b1 = 100;
byte b2 = -50;
// byte b3 = 200;  // 오류! 범위 초과
```

#### short

- **크기**: 2바이트 (16비트)
- **범위**: -32,768 ~ 32,767
- **용도**: 중간 크기 정수

```java
short s1 = 1000;
short s2 = -5000;
```

#### int (기본 정수형)

- **크기**: 4바이트 (32비트)
- **범위**: -2,147,483,648 ~ 2,147,483,647
- **용도**: 가장 많이 사용하는 정수형 (기본 타입)

```java
int number = 1000000;
int negative = -500000;
```

#### long

- **크기**: 8바이트 (64비트)
- **범위**: -9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807
- **용도**: 큰 정수
- **리터럴**: 숫자 뒤에 `L` 또는 `l` 붙임

```java
long bigNumber = 10000000000L;  // L 필수!
long another = 123L;
```

### 2. 실수형 (Floating-Point Types)

#### float

- **크기**: 4바이트 (32비트)
- **정밀도**: 약 7자리
- **용도**: 작은 실수
- **리터럴**: 숫자 뒤에 `F` 또는 `f` 붙임

```java
float f1 = 3.14F;  // F 필수!
float f2 = 1.5f;
float f3 = 100;     // 정수도 가능 (자동 변환)
```

#### double (기본 실수형)

- **크기**: 8바이트 (64비트)
- **정밀도**: 약 15자리
- **용도**: 가장 많이 사용하는 실수형 (기본 타입)

```java
double d1 = 3.14159;
double d2 = 1.5;
double d3 = 100;     // 정수도 가능
double d4 = 1.5D;    // D는 선택사항
```

### 3. 문자형 (Character Type)

#### char

- **크기**: 2바이트 (16비트, 유니코드)
- **범위**: 0 ~ 65,535 (유니코드 문자)
- **용도**: 단일 문자 저장

```java
char c1 = 'A';           // 작은따옴표 사용
char c2 = '한';
char c3 = 65;             // 'A'와 동일 (유니코드)
char c4 = '\u0041';       // 'A'와 동일 (유니코드 이스케이프)
char c5 = '\n';           // 줄바꿈 문자
char c6 = '\t';           // 탭 문자
```

#### 특수 문자 (이스케이프 시퀀스)

```java
'\n'  // 줄바꿈 (New Line)
'\t'  // 탭 (Tab)
'\r'  // 캐리지 리턴 (Carriage Return)
'\\'  // 백슬래시
'\''  // 작은따옴표
'\"'  // 큰따옴표
```

### 4. 논리형 (Boolean Type)

#### boolean

- **크기**: JVM에 따라 다름 (보통 1바이트)
- **값**: true 또는 false만 가능
- **용도**: 조건문, 반복문에서 사용

```java
boolean isTrue = true;
boolean isFalse = false;
// boolean b = 1;  // 오류! true/false만 가능
```

### 기본 데이터 타입 비교표

| 타입        | 크기    | 범위/값          | 기본값   | 용도             |
| ----------- | ------- | ---------------- | -------- | ---------------- |
| **byte**    | 1바이트 | -128 ~ 127       | 0        | 작은 정수        |
| **short**   | 2바이트 | -32,768 ~ 32,767 | 0        | 중간 정수        |
| **int**     | 4바이트 | 약 ±21억         | 0        | 일반 정수 (기본) |
| **long**    | 8바이트 | 약 ±922경        | 0L       | 큰 정수          |
| **float**   | 4바이트 | 약 ±3.4×10³⁸     | 0.0F     | 작은 실수        |
| **double**  | 8바이트 | 약 ±1.7×10³⁰⁸    | 0.0      | 일반 실수 (기본) |
| **char**    | 2바이트 | 0 ~ 65,535       | '\u0000' | 단일 문자        |
| **boolean** | 1바이트 | true/false       | false    | 논리값           |

### 기본값 (Default Values)

#### 클래스 변수와 인스턴스 변수

```java
public class DefaultValues {
    // 인스턴스 변수는 기본값으로 자동 초기화
    int number;        // 0
    double price;      // 0.0
    char letter;       // '\u0000'
    boolean flag;      // false
    String name;       // null
}
```

#### 지역 변수

```java
public void method() {
    // 지역 변수는 반드시 초기화 필요!
    int number;        // 초기화 안 됨
    // System.out.println(number);  // 오류!

    int count = 0;     // 초기화 필요
}
```

## 참조 데이터 타입 (Reference Types)

<details>
<summary>참조 타입의 개념과 종류</summary>

### 참조 타입이란?

- **정의**: 객체의 주소(참조)를 저장하는 타입
- **특징**: 실제 데이터는 Heap 영역에 저장되고, 변수는 그 주소를 저장
- **기본값**: null

### 참조 타입 종류

#### 1. 클래스 (Class)

```java
// String 클래스
String name = "Java";
String greeting = new String("Hello");

// 사용자 정의 클래스
Person person = new Person();
```

#### 2. 배열 (Array)

```java
int[] numbers = {1, 2, 3, 4, 5};
String[] names = new String[5];
```

#### 3. 인터페이스 (Interface)

```java
List<String> list = new ArrayList<>();
Map<String, Integer> map = new HashMap<>();
```

### 기본형 vs 참조형 비교

#### 메모리 저장 방식

```java
// 기본형: 값 자체 저장
int a = 10;
int b = a;      // 값 복사
b = 20;         // a는 여전히 10

// 참조형: 주소 저장
int[] arr1 = {1, 2, 3};
int[] arr2 = arr1;     // 주소 복사 (같은 객체 참조)
arr2[0] = 100;         // arr1[0]도 100으로 변경됨!
```

#### null 값

```java
// 기본형은 null 불가
// int number = null;  // 오류!

// 참조형은 null 가능
String name = null;
int[] array = null;
```

### String 클래스

#### String 특징

- **불변성**: 한 번 생성되면 변경 불가
- **String Pool**: 리터럴은 풀에서 재사용

```java
String str1 = "Hello";           // String Pool에 저장
String str2 = "Hello";           // 같은 객체 재사용
String str3 = new String("Hello");  // 새로운 객체 생성

System.out.println(str1 == str2);      // true (같은 객체)
System.out.println(str1 == str3);      // false (다른 객체)
System.out.println(str1.equals(str3)); // true (값은 같음)
```

### 배열 (Array)

#### 배열 선언과 초기화

```java
// 방법 1: 선언 후 초기화
int[] numbers;
numbers = new int[5];

// 방법 2: 선언과 동시에 초기화
int[] scores = new int[5];

// 방법 3: 리터럴로 초기화
int[] data = {1, 2, 3, 4, 5};

// 방법 4: 선언만
int[] arr = new int[]{10, 20, 30};
```

#### 배열 사용

```java
int[] numbers = {10, 20, 30, 40, 50};

// 인덱스로 접근 (0부터 시작)
System.out.println(numbers[0]);  // 10
System.out.println(numbers[4]);  // 50

// 배열 길이
System.out.println(numbers.length);  // 5

// 배열 순회
for (int i = 0; i < numbers.length; i++) {
    System.out.println(numbers[i]);
}
```

## 타입 변환 (Casting)

<details>
<summary>자동 타입 변환과 강제 타입 변환</summary>

### 타입 변환이란?

- **정의**: 한 데이터 타입을 다른 데이터 타입으로 변환하는 것
- **종류**: 자동 타입 변환(묵시적), 강제 타입 변환(명시적)

### 자동 타입 변환 (Implicit Casting)

#### 개념

- **정의**: 작은 타입에서 큰 타입으로 변환 시 자동으로 변환
- **방향**: 작은 타입 → 큰 타입
- **데이터 손실 없음**

#### 변환 규칙

```
byte → short → int → long → float → double
                ↑
               char
```

#### 예시

```java
// 작은 타입 → 큰 타입 (자동 변환)
byte b = 100;
int i = b;          // 자동 변환
long l = i;         // 자동 변환
float f = l;        // 자동 변환
double d = f;       // 자동 변환

// char → int
char c = 'A';
int code = c;       // 65 (자동 변환)

// int → double
int number = 100;
double decimal = number;  // 100.0 (자동 변환)
```

### 강제 타입 변환 (Explicit Casting)

#### 개념

- **정의**: 큰 타입에서 작은 타입으로 변환 시 명시적으로 변환 필요
- **방향**: 큰 타입 → 작은 타입
- **데이터 손실 가능**

#### 문법

```java
(변환할타입) 값
```

#### 예시

```java
// 큰 타입 → 작은 타입 (강제 변환)
double d = 3.14;
int i = (int) d;        // 3 (소수점 손실)
float f = (float) d;    // 3.14F

long l = 1000000L;
int num = (int) l;      // 강제 변환

// 주의: 범위 초과 시 잘못된 값
int big = 200;
byte b = (byte) big;    // -56 (오버플로우 발생!)
```

#### 데이터 손실 예시

```java
double pi = 3.14159;
int integer = (int) pi;  // 3 (소수점 손실)

long large = 3000000000L;
int small = (int) large;  // -1294967296 (오버플로우!)
```

### 연산 시 타입 변환

#### 산술 연산

```java
// 피연산자 중 하나가 double이면 결과도 double
int a = 10;
double b = 3.0;
double result = a + b;  // 13.0 (int가 double로 변환)

// 피연산자 중 하나가 long이면 결과도 long
int x = 100;
long y = 200L;
long sum = x + y;  // 300L (int가 long으로 변환)
```

#### 문자열 결합

```java
// + 연산자: 문자열이 하나라도 있으면 문자열 결합
String result1 = "Hello" + 123;      // "Hello123"
String result2 = 10 + 20 + "30";     // "3030" (10+20 먼저 계산)
String result3 = "10" + 20 + 30;      // "102030" (왼쪽부터)
```

### 타입 변환 주의사항

#### 1. 범위 초과

```java
int large = 300;
byte small = (byte) large;  // -44 (오버플로우)
```

#### 2. 정밀도 손실

```java
double precise = 3.14159265359;
float less = (float) precise;  // 정밀도 손실
```

#### 3. 참조 타입 캐스팅

```java
// 상속 관계에서만 가능
Animal animal = new Dog();
Dog dog = (Dog) animal;  // 가능

// 상속 관계가 아니면 ClassCastException
// String str = (String) animal;  // 오류!
```

## 연산자 (Operators)

<details>
<summary>산술, 비교, 논리, 비트 연산자 등</summary>

### 연산자 종류

Java는 다양한 연산자를 제공합니다:

1. **산술 연산자**
2. **비교 연산자**
3. **논리 연산자**
4. **비트 연산자**
5. **대입 연산자**
6. **증감 연산자**
7. **삼항 연산자**

### 1. 산술 연산자

#### 기본 산술 연산

```java
int a = 10;
int b = 3;

System.out.println(a + b);  // 13 (덧셈)
System.out.println(a - b);  // 7 (뺄셈)
System.out.println(a * b);  // 30 (곱셈)
System.out.println(a / b);  // 3 (나눗셈, 정수 나눗셈)
System.out.println(a % b);  // 1 (나머지)
```

#### 정수 나눗셈 주의

```java
int result = 10 / 3;        // 3 (소수점 버림)
double result2 = 10 / 3;     // 3.0 (정수 나눗셈 후 변환)
double result3 = 10.0 / 3;   // 3.333... (실수 나눗셈)
double result4 = (double) 10 / 3;  // 3.333... (명시적 변환)
```

### 2. 비교 연산자 (관계 연산자)

#### 비교 연산자 종류

```java
int a = 10;
int b = 20;

System.out.println(a == b);  // false (같음)
System.out.println(a != b);  // true (다름)
System.out.println(a < b);   // true (작음)
System.out.println(a > b);   // false (큼)
System.out.println(a <= b);  // true (작거나 같음)
System.out.println(a >= b);  // false (크거나 같음)
```

#### 참조 타입 비교

```java
String str1 = "Hello";
String str2 = "Hello";
String str3 = new String("Hello");

System.out.println(str1 == str2);      // true (같은 객체)
System.out.println(str1 == str3);      // false (다른 객체)
System.out.println(str1.equals(str3)); // true (값 비교)
```

### 3. 논리 연산자

#### 논리 연산자 종류

```java
boolean x = true;
boolean y = false;

System.out.println(x && y);   // false (AND, 둘 다 true여야 true)
System.out.println(x || y);   // true (OR, 하나라도 true면 true)
System.out.println(!x);       // false (NOT, 반대)
```

#### 단축 평가 (Short-Circuit Evaluation)

```java
// && 연산자: 앞이 false면 뒤는 평가 안 함
int a = 10;
if (a > 20 && a++ > 0) { }  // a++ 실행 안 됨
System.out.println(a);      // 10

// || 연산자: 앞이 true면 뒤는 평가 안 함
int b = 10;
if (b < 20 || b++ > 0) { }  // b++ 실행 안 됨
System.out.println(b);      // 10
```

### 4. 비트 연산자

#### 비트 연산자 종류

```java
int a = 5;   // 0101 (이진수)
int b = 3;   // 0011 (이진수)

System.out.println(a & b);   // 1 (AND: 0001)
System.out.println(a | b);   // 7 (OR: 0111)
System.out.println(a ^ b);   // 6 (XOR: 0110)
System.out.println(~a);      // -6 (NOT: 비트 반전)
System.out.println(a << 1);  // 10 (왼쪽 시프트: 1010)
System.out.println(a >> 1);  // 2 (오른쪽 시프트: 0010)
```

#### 비트 연산 활용

```java
// 짝수/홀수 판별
int number = 10;
if ((number & 1) == 0) {
    System.out.println("짝수");
}

// 2의 거듭제곱
int power = 1 << 3;  // 8 (2³)
```

### 5. 대입 연산자

#### 기본 대입

```java
int a = 10;  // 대입
```

#### 복합 대입 연산자

```java
int a = 10;

a += 5;   // a = a + 5 (15)
a -= 3;   // a = a - 3 (12)
a *= 2;   // a = a * 2 (24)
a /= 4;   // a = a / 4 (6)
a %= 4;   // a = a % 4 (2)
```

### 6. 증감 연산자

#### 전위/후위 증감

```java
int a = 10;

// 후위 증가 (값 사용 후 증가)
int b = a++;  // b = 10, a = 11

int c = 10;
// 전위 증가 (증가 후 값 사용)
int d = ++c;  // c = 11, d = 11
```

#### 예시

```java
int x = 5;
System.out.println(x++);  // 5 출력, x는 6
System.out.println(++x);  // 7 출력, x는 7

int y = 10;
int result = y++ + ++y;  // 10 + 12 = 22
```

### 7. 삼항 연산자 (조건 연산자)

#### 문법

```java
조건 ? 값1 : 값2
```

#### 예시

```java
int a = 10;
int b = 20;

// if-else를 간단하게 표현
int max = (a > b) ? a : b;  // 20

String result = (a > 0) ? "양수" : "음수 또는 0";

// 중첩 사용
int score = 85;
String grade = (score >= 90) ? "A" :
               (score >= 80) ? "B" :
               (score >= 70) ? "C" : "F";
```

### 연산자 우선순위

#### 우선순위 순서 (높은 순서대로)

1. **괄호** `()`
2. **단항 연산자** `++`, `--`, `!`, `~`
3. **산술 연산자** `*`, `/`, `%` → `+`, `-`
4. **시프트 연산자** `<<`, `>>`, `>>>`
5. **비교 연산자** `<`, `>`, `<=`, `>=`, `instanceof`
6. **등가 연산자** `==`, `!=`
7. **비트 연산자** `&` → `^` → `|`
8. **논리 연산자** `&&` → `||`
9. **삼항 연산자** `? :`
10. **대입 연산자** `=`, `+=`, `-=` 등

#### 예시

```java
int result = 10 + 20 * 3;        // 70 (곱셈 먼저)
int result2 = (10 + 20) * 3;     // 90 (괄호 먼저)
boolean flag = a > 0 && b < 10;   // &&는 비교 연산자보다 나중
```

## 상수 (final)

<details>
<summary>final 키워드와 상수 사용법</summary>

### final 키워드

#### 개념

- **정의**: 변수, 메서드, 클래스에 사용하여 변경을 막는 키워드
- **용도**:
  - 변수: 상수 선언
  - 메서드: 오버라이드 방지
  - 클래스: 상속 방지

### final 변수 (상수)

#### 지역 상수

```java
public void method() {
    final int MAX_COUNT = 100;  // 상수 선언
    // MAX_COUNT = 200;  // 오류! 변경 불가

    final String GREETING = "Hello";
    // GREETING = "Hi";  // 오류!
}
```

#### 인스턴스 상수

```java
public class Circle {
    private final double PI = 3.14159;  // 인스턴스 상수
    private final int radius;             // 생성자에서 초기화 가능

    public Circle(int radius) {
        this.radius = radius;  // 생성자에서 한 번만 초기화 가능
        // this.radius = 10;   // 오류! 이미 초기화됨
    }
}
```

#### 클래스 상수 (static final)

```java
public class Constants {
    // 클래스 상수 (대문자와 언더스코어 사용)
    public static final int MAX_SIZE = 1000;
    public static final String APP_NAME = "My Application";
    public static final double PI = 3.14159265359;

    // 사용
    public void method() {
        int size = Constants.MAX_SIZE;
        // 또는
        int size2 = MAX_SIZE;  // 같은 클래스 내에서는 클래스명 생략 가능
    }
}
```

### final 변수 초기화 시점

#### 1. 선언 시 초기화

```java
final int VALUE = 10;
```

#### 2. 생성자에서 초기화

```java
public class Example {
    final int number;

    public Example(int num) {
        this.number = num;  // 생성자에서 초기화
    }
}
```

#### 3. 초기화 블록에서

```java
public class Example {
    final int value;

    {
        value = 100;  // 초기화 블록
    }
}
```

### final 참조 변수

#### 객체 참조는 변경 불가, 객체 내용은 변경 가능

```java
final int[] numbers = {1, 2, 3};

// numbers = new int[]{4, 5, 6};  // 오류! 참조 변경 불가
numbers[0] = 10;  // 가능! 배열 내용은 변경 가능
```

### 상수 명명 규칙

#### 관례

- **대문자 사용**
- **단어 구분은 언더스코어(\_) 사용**
- **의미 있는 이름 사용**

```java
// 좋은 예
final int MAX_STUDENT_COUNT = 100;
final String DEFAULT_USER_NAME = "Guest";
final double PI = 3.14159;

// 나쁜 예
final int max = 100;  // 의미 불명확
final int m = 100;    // 너무 짧음
```

### final의 활용

#### 1. 매직 넘버 제거

```java
// 나쁜 예
if (score > 90) { }

// 좋은 예
final int EXCELLENT_SCORE = 90;
if (score > EXCELLENT_SCORE) { }
```

#### 2. 설정값 관리

```java
public class Config {
    public static final String DB_URL = "jdbc:mysql://localhost:3306/mydb";
    public static final int MAX_CONNECTIONS = 10;
    public static final int TIMEOUT_SECONDS = 30;
}
```

#### 3. 불변 객체 보장

```java
final String name = "Java";  // 문자열 불변 보장
final List<String> list = new ArrayList<>();  // 참조는 불변, 내용은 가변
```
